# ユーザーテンプレート

このディレクトリでは、Spec Workflowのデフォルトテンプレートを上書きするカスタムテンプレートを作成できます。

## カスタムテンプレートの使い方

1. **カスタムテンプレートファイルを作成**してください。このディレクトリに、上書きしたいデフォルトテンプレートと全く同じ名前のファイルを作成します:
   - `requirements-template.md` - 要件定義書テンプレートを上書き
   - `design-template.md` - 設計書テンプレートを上書き
   - `tasks-template.md` - タスクドキュメントテンプレートを上書き
   - `product-template.md` - プロダクトステアリングテンプレートを上書き
   - `tech-template.md` - 技術ステアリングテンプレートを上書き
   - `structure-template.md` - 構造ステアリングテンプレートを上書き

2. **テンプレートの読み込み優先順位**:
   - システムはまずこの `user-templates/` ディレクトリを確認します
   - ここで一致するテンプレートが見つかった場合、それが使用されます
   - 見つからない場合は、`templates/` のデフォルトテンプレートが使用されます

## カスタムテンプレートの例

カスタム要件定義書テンプレートを作成するには:

1. このディレクトリに `requirements-template.md` というファイルを作成します
2. カスタム構造を追加します。例:

```markdown
# 要件定義書

## エグゼクティブサマリー
[カスタムセクション]

## ビジネス要件
[カスタム構造]

## 技術要件
[カスタムフィールド]

## カスタムセクション
[ワークフローに固有のセクションを追加]
```

## テンプレート変数

テンプレートには、ドキュメント作成時に置換されるプレースホルダーを含めることができます:
- `{{projectName}}` - プロジェクト名
- `{{featureName}}` - 仕様化する機能の名前
- `{{date}}` - 現在の日付
- `{{author}}` - ドキュメントの作成者

## ベストプラクティス

1. **デフォルトから始める**: `../templates/` のデフォルトテンプレートをコピーして出発点にする
2. **構造の一貫性を保つ**: ツールの互換性のため、類似したセクションヘッダーを維持する
3. **変更をドキュメント化**: セクションを追加/変更した理由をコメントで説明する
4. **バージョン管理**: カスタムテンプレートをバージョン管理で追跡する
5. **十分にテストする**: カスタムテンプレートがspecワークフローツールで動作することを確認する

## 注意事項

- カスタムテンプレートはプロジェクト固有であり、パッケージ配布には含まれません
- `templates/` ディレクトリには各バージョンで更新されるデフォルトテンプレートが含まれています
- このディレクトリのカスタムテンプレートはアップデート時に保持されます
- カスタムテンプレートにエラーがある場合、システムはデフォルトテンプレートにフォールバックします
