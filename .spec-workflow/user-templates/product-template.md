# 製品概要 (Product Overview)

## 製品の目的
本製品は、案件ごとに複数の候補日を管理し、最適な日程を確定させるためのスケジュール管理アプリです。「調整中の予定」を最大5つまで保持できる柔軟性と、確定時に不要な候補を自動掃除するスマートな操作性を提供し、少人数チームにおけるスケジューリングの混乱を解消します。

## ターゲットユーザー
- **主なユーザー**: 10名程度の小規模なエンジニアチームやプロジェクトメンバー。
- **ペインポイント**:
    - 予定が確定するまでカレンダーが仮予定で埋まり、空き時間が分かりにくい。
    - 確定後に残った「ボツ候補日」を消し忘れて、ダブルブッキングの原因になる。
    - 複数の案件が重なり、特定の日付に業務が集中していることに気づけない。

## 主要機能

1. **マルチ候補日登録**: 1つの予定に対し、最大5件までの候補日をクイックに登録・変更・削除で管理する機能。
2. **オートクリーン確定**: 1つの候補を「確定」とした瞬間、他の候補日をシステムが自動的に削除し、データを常に最新・清潔に保つ機能。
3. **オーバーロード（過密日）検知**: 全案件を横断し、過密日の条件が満たされた場合は過密日一覧に表示する
4. **レスポンシブ・マルチデバイス対応**: Firebase Auth によるセキュアな認証をベースに、PCとスマホのどちらからでも快適に操作できるUI。
5. **カレンダー表示**: 縦軸を日付、横軸を案件にしたGoogleカレンダーのようなUI表示も可能でカレンダーのところから予定を登録することができる。当日は青色一列で表示、過密日は赤一列で表示
6. **過密日の設定項目**: 過密日として扱う条件を設定することができる。1日に指定した工数を超過した場合過密日として扱う。設定項目は過密日として扱う工数、確定日の工数、調整日の工数が設定できる
7. **タグ管理**: 撮影。打ち合わせなどのタグを作成や編集、削除できる。

## ビジネス目標
- チーム内のスケジュール調整にかかるコミュニケーションコストを削減する。
- 候補日の自動クリーン機能により、カレンダーの「ゴミデータ」を排除し、チームの予定把握精度を向上させる。
- 過密日の可視化により、無理なスケジューリングを防ぎ、チームの稼働品質を維持する。
